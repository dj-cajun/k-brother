# 코드 검토 보고서 (Node.js/Express 백엔드)

## 보안 취약점

1. **JWT 검증 누락**:
   - `/api/auth/login` 및 `/api/merchant/login` 엔드포인트에서 JWT 서명 알고리즘을 명시하지 않음
   - `jwt.verify()` 호출 시 `algorithms` 옵션 누락으로 HMAC-SHA256 외의 알고리즘 허용 가능성

2. **SQL Injection 위험**:
   - 동적 쿼리 생성 시(`/api/merchant/transactions`) 직접 문자열 연결 사용
   - `params.length` 기반으로 동적 파라미터 할당하는 방식은 취약할 수 있음

3. **세션 관리 문제**:
   - 리프레시 토큰이 구현되었지만 실제 사용/갱신 로직 누락
   - JWT 토큰 갱신 메커니즘 없음

4. **민감 정보 노출**:
   - 오류 메시지에서 시스템 정보(예: DB 오류)가 클라이언트에 노출될 수 있음
   - `console.error`에 민감한 정보 기록 가능성

5. **Rate Limiting 부족**:
   - 로그인/회원가입 엔드포인트에 대한 별도의 rate limiting 없음

## 기능적 결함

1. **트랜잭션 관리**:
   - `/api/merchant/redeem`에서 `client.release()`가 `finally` 블록에 있지만 롤백 후에도 호출됨
   - 일부 쿼리에서 트랜잭션 사용하지 않음(예: `/api/auth/signup`의 쿠폰 발행)

2. **쿠폰 상태 관리**:
   - 쿠폰 상태 변경 시 검증 로직 부족(`ISSUED` → `USED` 전환만 확인)

3. **Redis Lua 스크립트**:
   - `ATOMIC_CONSUME_SCRIPT`에서 `KEYS[1]` 대신 `KEYS[1]` 사용(문법 오류)
   - 스크립트에 오류 처리 로직 없음

4. **날짜 처리**:
   - 타임존을 고려하지 않은 날짜 비교(`from`/`to` 파라미터 처리)

5. **페이징 처리**:
   - `/api/merchant/transactions`에서 `page`와 `limit` 파라미터 유효성 검사 없음

## 코드 품질 문제

1. **에러 처리**:
   - 일관되지 않은 오류 응답 형식
   - 중복된 오류 처리 코드(예: JWT 검증)

2. **환경 변수**:
   - 필수 환경 변수에 대한 검증 누락(예: `JWT_SECRET`, `DATABASE_URL`)

3. **리소스 관리**:
   - PostgreSQL 커넥션 풀에서 커넥션 릴리스 누락 가능성
   - Redis 연결 상태 검사 없음

4. **로그 관리**:
   - 구조화된 로깅 시스템 없음
   - 민감한 정보가 로그에 기록될 수 있음

5. **테스트 용이성**:
   - 의존성 주입 패턴 없어 단위 테스트 어려움

## 개선 제안

1. **보안 강화**:
   ```javascript
   // JWT 검증 시 알고리즘 명시
   jwt.verify(token, process.env.JWT_SECRET, { algorithms: ['HS256'] });
   ```

2. **SQL Injection 방지**:
   ```javascript
   // 동적 쿼리 대신 정적 쿼리 사용
   const whereClauses = [];
   if (from) whereClauses.push('t.redeemed_at >= $2');
   if (to) whereClauses.push('t.redeemed_at <= $3');
   const where = whereClauses.length ? 'AND ' + whereClauses.join(' AND ') : '';
   ```

3. **에러 처리 통일**:
   ```javascript
   // 공통 에러 핸들러 미들웨어 추가
   app.use((err, req, res, next) => {
     if (err.name === 'JsonWebTokenError') {
       return res.status(401).json({ error: 'INVALID_TOKEN' });
     }
     // 다른 에러 유형 처리
   });
   ```

4. **환경 변수 검증**:
   ```javascript
   // 시작 시 필수 환경 변수 확인
   const requiredEnvVars = ['JWT_SECRET', 'DATABASE_URL'];
   requiredEnvVars.forEach(env => {
     if (!process.env[env]) throw new Error(`Missing required env var: ${env}`);
   });
   ```

5. **로깅 개선**:
   ```javascript
   // 구조화된 로깅 도입
   const logger = require('pino')({
     level: process.env.LOG_LEVEL || 'info',
     redact: ['req.headers.authorization', 'req.body.password']
   });
   ```

## 결론

현재 코드는 기본적인 기능 요구사항을 구현하고 있지만, 프로덕션 환경에서 사용하기에는 여러 보안 및 안정성 문제가 존재합니다. 특히 인증/인가 처리, 데이터 유효성 검사, 오류 처리 부분에서 상당한 개선이 필요합니다. 모든 외부 입력에 대한 엄격한 유효성 검사 추가와 핵심 비즈니스 로직에 대한 더 강력한 트랜잭션 관리가 필요합니다. 또한 보안 취약점을 해결하기 위해 JWT 처리 방식을 개선하고 모든 데이터베이스 쿼리를 파라미터화해야 합니다.
